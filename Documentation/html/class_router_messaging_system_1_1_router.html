Detailed Description

Router Messaging System for Unity Engine.

A replacement messaging system for Unity Engine built around not using blasted strings.

Author
Philip Muzzall
Version
1.0.0
Date
3/17/2014
Copyright
Property of Philip Muzzall

Each Component must register with the Router before it can receive messages.
To do this create a Route and register it with the Router using Router.AddRoute(Route NewRoute).
Components can register multiple Routes with the Router to subscribe to multiple events.

Note
Router internally maintains routing tables for all registered Routes.
These tables do not exists until the first Route has been registered.
If all Routes are removed then Router will destroy these tables and recreate them again when they are needed.

static void RouterMessagingSystem.Router.FlushRoutes ( )
static

Flushes all Routes from the routing table.

Warning
Only use this if you absolutely need to reset the entire routing table at runtime.
Every subscriber will need to re-register with the Router after the tables are flushed.
This is a relatively slow operation, depending on the amount of subscribers that need to be re-added.
